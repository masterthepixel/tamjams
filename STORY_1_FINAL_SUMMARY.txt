ARCHITECTURE NOTES
================================================================================

STORAGE PATTERN:
- Video URLs stored in product.metadata.videoUrls
- Stored as stringified JSON array (same pattern as nutrition facts)
- Undefined if product has no videos (graceful degradation)

SERIALIZATION:
videoUrls: product.videoUrls ? JSON.stringify(product.videoUrls) : undefined

DATA FORMAT IN API:
{
  "metadata": {
    "videoUrls": "[\"\/static\/strawberry-demo.mp4\",\"\/static\/strawberry-usage.webm\"]"
  }
}

KEY BENEFITS:
✓ No database schema changes
✓ Uses existing Medusa metadata field
✓ Fully backwards compatible
✓ Consistent with nutrition field pattern
✓ Flexible (supports relative and absolute URLs)
